public class Human {
        private String firstName;
        private String lastName;

        @Override
        public boolean equals(Object o) {
            if (this == o) return true;
            if (o == null || getClass() != o.getClass()) return false;
            Human human = (Human) o;
            return Objects.equals(firstName, human.firstName) && Objects.equals(lastName, human.lastName);
        }

        @Override
        public int hashCode() {
            return Objects.hash(firstName, lastName);
        }

        // Старый стиль
        @Override
        public int hashCode() {
            int result = firstName != null ? firstName.hashCode() : 0;
            result = 31 * result + (lastName != null ? lastName.hashCode() : 0);
            return result;
        }
    }

POJO и коллекции обрабатываются так же само!

Правила equals:

Рефлексивность: x.equals(x) возвращает true.
Симметричность: Если x.equals(y) возвращает true, то и y.equals(x) вернёт true.
Транзитивность: Если x.equals(y) и y.equals(z) возвращают true, то и x.equals(z) вернёт true.
Консистентность: Если объекты x и y не изменяются, то вызовы x.equals(y) должны возвращать один и тот же результат.
Неравенство null: x.equals(null) возвращает false

Правила hashCode():

Согласованность: Если объекты равны по equals() - их хэш-коды тоже равными.
Но не наоборот! (коллизия)
Повторяемость: hashCode() Для одного и того же объекта возвращает одно и то же, если объект не изменялся.
