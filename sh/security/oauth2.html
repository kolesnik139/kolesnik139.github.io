OAuth 2.0

Протокол АВТОРИЗАЦИИ, позволяющий выдать одному сервису (приложению) права на доступ к ресурсам пользователя на другом сервисе
0. Разработчик регистрирует приложение (Клиент) в Google Console (Сервис) и получает Client Id и Client Secret
1. Клиент формирует GET запрос на Сервис
client_id
redirect_url
scope - список разрешений
response_type=code
state - UNIQUE_RANDOM_STRING
2. При переходе по ссылке открывается окно Сервиса, Владелец логинится и соглашается предоставить права
3. Сервис перенаправляет ответ на redirect_url и в параметре передаёт AUTHORIZATION_CODE и возвращает state
4. Клиент отправляет POST запрос
client_id
client_secret
redirect_uri
code=AUTHORIZATION_CODE
grant_type=authorization_code
и получает JWT Access Token и Refresh Token

JWT (JSON Web Token)

Header: {"alg": "HS256", "typ": "JWT"}
Payload: {"sub": "1234567890", "name": "John Doe", "iat": 1516239022}
Signature: SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c
