Позволяет запрашивать именно те данные которые нужны (можно связные), и получить их в одном запросе
Есть один эндпоинт для всех запросов, и схема API на стороне сервера

Виды запросов:
Query
Mutation
Subscription

Схема: 
    type Character {
      name: String
      # ... Другие поля
      friends: [Character]
    }

    type Query {
      hero: Character
    }

Запрос: 
    query {
      hero {
        name
        friends {
          name
        }
      }
    }

Ответ: 
  {
    "data": {
      "hero": {
        "name": "RD-D2",
        "friends": [
          {
            "name": "Luke Skywalker"
          },
          {
            "name": "Han Solo"
          }
        ]
      }
    }
  }
